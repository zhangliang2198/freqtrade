# 在特性分支上做二次开发

```bash
# 1) 基于最新上游分支创建你的特性分支
git checkout develop
git pull --rebase origin develop
git checkout -b feature/my-cool-change

# 2) 正常写代码、提交
# ... edit files ...
git add -A
git commit -m "feat: my cool change"

# 3) 随时把特性分支推到你的 fork
git push -u origin feature/my-cool-change
```

# 当上游更新时，给特性分支「续命」

```bash
# 在任何时候
git fetch upstream
git checkout develop
git rebase upstream/develop           # 或 merge

# 回到你的特性分支
git checkout feature/my-cool-change

# 方式一：把特性分支「变基」到最新的 develop（历史直线）
git rebase develop
# 如果有冲突，按提示解决后： git add <冲突文件> && git rebase --continue

# 方式二：直接把最新 develop 合并进来（简单）
# git merge develop

# 推回远端（若用了 rebase，需要强推当前分支）
git push -f origin feature/my-cool-change
```

# 发 PR 的两种路径

```bash
对上游发 PR

目标仓库：freqtrade/freqtrade

源分支：YOURNAME/freqtrade:feature/my-cool-change

目标分支：freqtrade/freqtrade:develop（或上游默认分支）

只在自己 fork 内部维护

以后继续按上面「同步上游 → 变基/合并 → 继续开发」的节奏走。
```
